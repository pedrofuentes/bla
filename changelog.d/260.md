- Snippet trigger matching (M4, issue #260, AC-52): a new pure
  `snippets::match_snippet(text, snippets)` resolves whether a transcript
  contains a configured snippet's trigger phrase and, if so, returns that
  snippet's stored body — first match in the caller-supplied list order
  wins (mirroring `context::resolve_tone_for_app`'s AC-40 precedent), and
  matching is case-insensitive and word-boundary-anchored (a trigger like
  "sig" matches "add my sig now" but not "signature" — the documented
  match-boundary decision AC-52 left to the implementer, reusing the same
  `\b` word-boundary primitive `cleanup.rs`'s filler-word regex already
  relies on). Reuses `store::Snippet` (#258) directly rather than
  redefining it. Pure logic only, table-driven synthetic-fixture tests —
  no pipeline/IPC wiring (that's #263) and no store/schema change.
