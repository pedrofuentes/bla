- Selection capture/replace for command mode (M4, issue #257, AC-48): a new
  `CopySynthesizer` trait (real impl `EnigoCopy`, via `enigo`) mirrors the
  existing `PasteSynthesizer`/`EnigoPaste` one key over — Cmd+C / Ctrl+C
  instead of Cmd+V / Ctrl+V, sharing the same platform-modifier lookup
  (`paste_modifier`). Two new pure functions reuse `output.rs`'s existing
  clipboard-swap machinery: `capture_selection` saves the pre-copy clipboard,
  synthesizes copy, and returns the captured selection (wrapped in the
  no-log `ClipboardPayload`, ADR-0003) without touching the clipboard again;
  `replace_selection` writes the transformed text, synthesizes paste, and
  restores the clipboard afterward — but restores the ORIGINAL pre-copy
  value captured earlier, not the intermediate captured-selection value that
  sat on the clipboard while the instruction was being recorded and
  transformed, with the same skip-on-change safety `should_restore_clipboard`
  already provides for AC-9. `CapturedSelection`, the struct threading both
  values between the two functions, is added to the module's compile-time
  no-`Debug`/`Display`/`Serialize` guard alongside `ClipboardPayload`.
  Logic-only, exercised with fake `Clipboard`/`CopySynthesizer`/
  `PasteSynthesizer` doubles — no hotkey or pipeline wiring yet (that's
  #259).
