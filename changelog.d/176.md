- Fixed overlapping dictations clobbering each other's pipeline state (issues #176, #175, #174): starting a new dictation while a previous one is still transcribing could have the earlier dictation's stale completion overwrite the new one's live state — dropping its recording-pill waveform mid-dictation, showing the wrong pill chrome for up to 1.5s, or hiding the pill out from under a still-live settle. A per-dictation generation id (minted at `StartRecording`, mirroring the existing pill-visibility-epoch pattern) is now threaded through the background transcription thread and every settle it can spawn; a completion whose generation no longer matches the live dictation now no-ops entirely (no state write, no event emit, no settle thread) instead of clobbering the newer dictation. The settle thread's delayed pill-hide also now locks `pipeline_state` before reading the epoch/generation atomics (rather than after), closing a narrow TOCTOU window that could hide the pill one tick early during two overlapping settles.
