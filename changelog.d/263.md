- Wire snippet matching into the dictation pipeline (M4, issue #263, AC-53):
  `Pipeline::run` now checks `snippets::match_snippet` (#260) against the RAW
  transcript BEFORE ever consulting the configured `Cleanup` — on a match, it
  short-circuits straight to `output::route` with the snippet's stored body
  and skips `Cleanup::clean` entirely (both the `RegexCleanup` baseline and
  any LLM pass) for that dictation. `PipelineOpts` gained a `snippets` field
  and `Outcome` a `snippet_matched` flag; `run_pipeline_in_background`
  (`lib.rs`) reads `Store::list_snippets()` fresh per dictation (never
  cached, best-effort like the existing dictionary/tone reads) and feeds it
  through, mirroring the dictionary/tone wiring pattern. Per #242's
  cofounder-approved M4 scope, matching is against the raw transcript, not
  cleaned text — see this PR's description for a flagged conflict with
  PRD.md's older AC-24 wording, which describes the opposite (post-cleanup)
  ordering and needs a docs correction or cofounder override.
