- Snippets store backend (M4, issue #258, AC-51): a new `snippets` table
  (migration 4 on the `Store` foundation from #161/#198) mapping a trigger
  phrase to the body text it expands to, with case-insensitive trigger
  uniqueness (`UNIQUE COLLATE NOCASE`, mirroring `dictionary.term`'s /
  `tone_rules.app_pattern`'s existing precedent) and CRUD
  (`Store::add_snippet`/`list_snippets`/`update_snippet`/`remove_snippet`).
  Unlike `tone_rules`' upsert-by-pattern, snippets use an explicit by-id
  `update_snippet` for edits; the schema's uniqueness constraint still
  rejects an update that would collide with a different row's trigger.
  Store-only — no matching logic (#260) and no IPC/UI (#261) yet.
