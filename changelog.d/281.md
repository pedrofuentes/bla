- Fixed a command-mode hotkey bug (issue #281, ac7-p0) where holding a
  letter-key chord (e.g. `Ctrl+Shift+O`) in hold-to-record mode leaked
  repeated keystrokes into the focused app — typing over the selected text
  *before* the command transform ran, so the transform saw the leaked text
  instead of the real selection. Neither macOS's Carbon hotkey registration
  nor Windows' `RegisterHotKey` can suppress a keydown while a chord is
  held, so instead of trying to suppress the leak, the command-mode
  hotkey's trigger key is now constrained to a function key (F1-F24) — a
  function key produces no text character, so a leaked keydown can no
  longer clobber a selection. Enforced both live in the settings window's
  picker and at the `set_settings` persistence boundary, with a clear
  inline explanation ("Command mode needs a function key..."). Deliberately
  does not change the dictation hotkey's validation (follow-up: #291).
- Fixed issue #292 (follow-up from #281): the shipped default
  `command_hotkey` was still `Control+Shift+C` — a letter key, exposing
  every fresh install (and any user who never customized the field) to
  #281's exact harm out of the box. The default is now
  `Control+Shift+F12` — a function key on every physical keyboard
  (including laptops with no F13+ key), low-collision, and still fully
  user-rebindable in the settings window.
